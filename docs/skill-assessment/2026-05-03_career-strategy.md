# キャリア戦略メモ — 2026-05-03

## 前提

- コード知識ゼロからAI（Cursor / Claude）と協働して開発してきた
- 副業禁止の会社に所属
- 構築したシステムは自分のアカウント（GCP / Supabase / Google Workspace）に紐づいており、そのままの納品は困難

---

## スキル構成の特徴：普通のエンジニアと逆転している

| 普通の学習順序 | 自分の学習順序 |
|---|---|
| 1. 文法・基礎 → 2. テスト → 3. 設計 → 4. 運用 | 1. 課題発見 → 2. 設計 → 3. 運用 → 4. 文法・テストは後回し |

- 高く評価された部分は「コードの書き方」ではなく**システムの振る舞いについての思考力**
  - Web API に処理を持たせない判断
  - dry-run → apply で事故を防ぐ設計
  - E32 で止めて E40 は controller が呼ぶ制御構造
  - デプロイ障害を経験し README にアンチパターンとして記録
- 低く評価された部分は**エンジニアの常識**（テスト、CI、コミット規律、重複排除）
- これは**知識の問題であって能力の問題ではない**。穴を埋めるのは、ゼロから設計力を身につけるよりずっと簡単

## ココナラ案件の現実的な距離感

### 技術的には受けられる案件が既にある

- Google Apps Script / スプレッドシート自動化
- PDF 処理・データ抽出の自動化
- Supabase / Firebase を使った簡易 Web アプリ
- Gemini API を組み込んだ業務ツール

### 問題：納品しにくい構造

現在のシステムは全て自分の環境に依存している。

```
settings.py → .env → 自分の Supabase URL / Key
cloudbuild.yaml → 自分の Artifact Registry / Cloud Run
Gmail 連携 → 自分の OAuth トークン
Drive 連携 → 自分のサービスアカウント
```

他人に納品するには環境の付け替えが必要で、その作業が本体開発より重くなる可能性がある。

### 売り方の方向性

| 方向 | 内容 | 初期コスト | メンテ負荷 |
|---|---|---|---|
| A: スクリプト納品 | 環境非依存のツールを切り出して売り切り | 切り出し作業が必要 | なし |
| B: 処理代行 | 自分の環境で動かして結果だけ納品 | ほぼゼロ | 案件ごと |

## 副業禁止下での実績作り

### ココナラの場合

- 最低出品価格 500 円。「実績作りのため格安」出品者は多い
- 年間雑所得 20 万円以下なら確定申告不要（給与所得者）、ただし住民税申告は必要
- 収入がほぼゼロでも「副業」に該当するかは会社の定義次第 → グレー

### より安全な代替手段（収入ゼロ = 副業に該当しない）

**1. OSS コントリビュート**
- GitHub の PDF 処理や AI 関連プロジェクトに PR を出す
- 収入ゼロ、完全合法
- エンジニアとしての実績はココナラの★5 より評価が高い

**2. 知人・友人の課題を無償で解決**
- 「PDF 整理が面倒」「紙の書類をデータ化したい」などの身近な課題
- 成果物をポートフォリオに載せる許可だけもらう

**3. Qiita / Zenn に技術記事を書く**
- パイプライン設計判断、Gemini API の使い方、pdfplumber + OpenCV の組み合わせ
- 試行錯誤の知識がそのまま記事になる
- 案件の引き合いが向こうから来ることもある

**4. GitHub でデモ用リポジトリを公開**
- アカウント情報を除いた汎用版を1つ公開
- 「この人はこれが作れる」の証明

## 推奨ルート

今は稼がなくていい。**「見せられるもの」を作ることに集中する。**

1. パイプラインのコア部分（PDF → 構造化データ）をスタンドアロンで切り出して **GitHub 公開**
2. 設計判断や試行錯誤を **Zenn に 2〜3 本記事を書く**
3. 副業禁止が解除された / フリーランスに転向するとき、**即座に受注できる状態**になっている

ココナラの★5 を 500 円で 3 件集めるより、GitHub 公開リポジトリ + 技術記事 2 本のほうが、長期的な資産としての価値は高い。
