# 📄 Document Management System (統合ドキュメント＆ライフハック管理システム)

本プロジェクトは、各種ドキュメント（PDF、画像等）の高度なAI解析にとどまらず、家計簿管理、カレンダー連携、PDF編集、さらには求人検索や生活必需品の比較環境までを包含した、**非常に巨大かつ高度な統合ライフハック・プラットフォーム**です。

全20以上のマイクロサービスがそれぞれの責務を持ちつつ、シームレスにデータベース(Supabase)を介して連携しています。各アプリに込められた「こだわり」や「高度なアルゴリズム」は、市販のSaaS製品にも匹敵するか、それ以上のカスタマイズが施されています。

以下に、稼働している各アプリケーションの**具体的な役割と、そこに実装されている極めてニッチかつ強力な工夫**を一つ残らず解説します。

---

## 📑 1. ドキュメント解析・ナレッジハブ (Core Pipeline)

本システムの心臓部とも言える「書類を読んで理解し、検索可能にする」ためのコアアプリ群です。

### `doc-processor` / `shared/pipeline` (AIドキュメント処理エンジン)
Web APIは「リクエストのキューイングのみ」を行い、真の重い処理は非同期のWorkerレイヤーとして実行される超堅牢なドキュメントパーサーです。
* **💡 恐るべき工夫: Stage A〜Kの完全防備なパイプライン**
  * **二重読み取り防止 (LayerPurge)**: OCRとAI（Gemini）が同じテキストを重複して読んでしまうのを防ぐため、Stage Bにてシステムが認識済みの一部テキストを画像レイヤー上で「白塗り（パージ）」してから次の工程に渡すという泥臭くも確実な前処理を実装しています。
  * **「表崩れ」の完全制圧 (Stage D)**: ベクトル線とラスター線の両方を検知し、エクセルのような表（セルマップ）を完全に復元。セル単位で画像を分割してモデルに渡すことで、LLM特有のハルシネーションやレイアウト崩壊を完全にシャットアウトします。
  * **文字密度でのLLM動的ルーティング (Stage E)**: 画像内の文字密度を計算し、文字が少ない地の文は高速な「Gemini Flash-lite」、情報が密な表部分は推論力が高い「Gemini Flash」へとリクエストを自動で切り替え、コストと精度の最高効率を叩き出しています。

### `doc-review` & `doc-search` (目視レビューと横断検索)
AIが作成した `ui_data` (フロントエンドが計算不要で即座に表示できるJSON形式) を即座に描画し、人間のオペレータが最終確認（レビュー）してから本登録を行い、それを強力に検索するためのUI群です。

## 🥪 2. PDF & ドキュメント加工ツール群

既製品のPDFツールでは痒い所に手が届かない課題を、独自の処理で解決しているツール群です。

### `json-pdf-embedder` & `ocr-editor`
スキャンした請求書などの画像をAIで読み取り、その結果を「見えないデータ」としてPDFに埋め込むアプリです。
* **💡 サンドイッチPDFの発明**: PyMuPDFの `render_mode=3` の仕様をハックし、人間の目に見える元の書類のレイアウトを一切崩すことなく、ページの上部に「機械（NotebookLM等）が100%正確に読み取れる透明なメタデータタグ」を精巧に埋め込んでいます。ページの回転状態（縦横）などを自力で計算・補正して正確な座標に描画する計算ロジックが組み込まれています。
* **実務最強の3ペインUI**: 画面を「元ファイルサムネイル」「プレビュー」「構造化データ」の3つに分割。Undo/Redo機能や一時保存機能を完備し、実際のオペレーターが「どこを修正すべきか」を一瞬で把握できる極上の手触りを実現しています。

### `pdf-splitter` (PDF見開き分割ツール)
見開きのスキャンPDFを、中央で「左」「右」のページに真っ二つに分割するツールです。
* **💡 画質の劣化ゼロ分割**: 単純に画像を半分にトリミングして再圧縮するのではなく、PyMuPDFの `mediabox` と `cropbox` の座標だけを操作する（Byte streamを共有しつつメタデータだけを弄る）アプローチを採用。これによりオリジナルの解像度やテキストデータを一切損なわずに一瞬で分割を完了します。「右から左(漫画や日本の縦書き)」順序にも対応しています。

### `html-to-a4` (HTML超最適化A4印刷機)
どんなHTMLフォーマットの画面でも、「必ず完璧なA4サイズのPDF・印刷プレビュー」に落とし込むための特殊コンバーターです。
* **💡 強制CSSインジェクション**: アップロードされたHTMLの `<head>` に、印刷時のマージン設定(`margin: 15mm !important;`)やテーブルの強制固定レイアウト、さらには指定クラス(`page-break-before: always`)での強制改ページなど、レイアウト崩れを許さないCSSを動的に注入（Inject）。画面の右下にはどんなページでも追従する「📄 A4でPDF化する」フローティングボタンを自動生成します。

## 🏦 3. 財務・家計・ライフスタイル管理

単なる「記録」にとどまらず、アルゴリズムによる自動化の恩恵を最大限に受けるアプリ群です。

### `kakeibo` / `kakeibo_view` (完全自動化・家計簿＆財務システム)
Pythonで3000行以上書かれている超本格的な財務管理システムで、一般的なSaaS家計簿アプリの限界を突破しています。
* **💡 APIキャッシュスキップと高速化**: SupabaseのPostgREST APIを直叩きする際、スキーマキャッシュの遅延を回避するために `Prefer: return=minimal` 仕様を用いて最速でUpsertを行うなど、パフォーマンスへの執念が詰まっています。
* **複雑怪奇な「消込」アルゴリズム**: 複数口座を跨ぐ資金移動やカード引き落としを相殺する機能（消込）。レガシーデータ（作成時間ベース）と最新データ（グループIDベース）の両方をパースし、左側・右側のトランザクションを見事にペアリング。
* **ローン・立て替えの徹底管理**: 普通の家計簿では管理が難しい「カードローン」や「利息」に専用データベースとダッシュボード(`loans`)を用意。
* **ルールベースの自動分類**: 文字列の完全一致・部分一致のみならず、「__INST_ONLY__」のような特殊プレフィックスによる口座名専用マッチングや、プラスマイナス(`amount_sign`)による条件分岐など、強力な正規表現/文字列マッチングエンジンを自作しています。

### `netsuper-search` (ネットスーパー横断検索)
Streamlitで構築。日々の買い物を最適化するため、複数のネットスーパー（ライフ、イオン等）の在庫と価格を即座に横断検索・比較し、最もお得かつ手間のない買い物をアシストします。

### `my-calendar-app` & `calendar-register` (AIイベント自動抽出＆共有カレンダー)
大量の学校プリントなどを画像やテキストで投げるだけで、スケジュールを全自動で解析・登録します。
* **💡 LLMの「おせっかい」をねじ伏せるプロンプト構成**: AIが気を利かせて「(4限弁当)」という注意書きを予定の場所や詳細に勝手に分離してしまう悪癖を、徹底的なプロンプトエンジニアリングで封殺。正確なスケジュール文字列として保持させます。
* **全量表示カスタマイズ (`my-calendar-app`)**: 一般的なカレンダーUI（Next.js環境）でイベントが多くなると発生する「+3件」などの見落とし原因の省略挙動を撤廃。1日の枠内で「ママ」などのメンバーごとの予定が何件あろうと全て表示しきる、実用性全振りのUIカスタマイズが行われています。

## 🧹 4. デジタル断捨離・クローラー・分析系

溜め込まれた無駄なデータを安全に掃除し、必要な情報を外から集めてくるユーティリティ群。

### `drive-duplicate-checker` & `drive-manager-flask` (ストレージ大掃除システム)
Google Drive等で無限に増殖した重複ファイル・フォルダを、絶対に安全に整理する特化ツールです。
* **💡 「ただの重複」を見分ける賢さ**: 同一ハッシュ（中身が同じ）でも、拡張子が `.IFO` と `.BUP` などで異なる場合は「システム上両方必要なファイル」と判断して消去対象から外す高度なセーフティ機能を搭載。
* **世代管理の自動認識**: ファイル名の「カッコ内の数字(例: `file (1).txt`)」などの記法揺らぎを吸収。UI上の「Mark Older Files」ボタン一つで、同一ファイルの最新版だけを保護し、古いバージョンを何千件でも一瞬で削除候補にマークします。
* **フォルダ統合（Merge）推論力**: ファイル単位だけでなく、「フォルダの総サイズ」と「内部の部分的なMD5ハッシュ」による独自のフィンガープリントを計算。AとBのフォルダが「統合しても問題ないほど類似しているか」をスコアリングして提案します。フロントエンドにはフリーズ対策のProgressUIも完備。

### `doda-scraper` & `doda-dashboard`
求人サイト（Dodaなど）に特化したスクレイピングエンジンと、その取得結果を分析するダッシュボード。市場の傾向や自身のスキルマッチングなどをデータドリブンに解析します。

### その他 (`ai-cost-tracker`, `daily-report`, `data-ingestion` 等)
* システム全体のGemini API等のトークン消費やインフラコストを監視し、無駄遣いを検知するトラッキング。
* その日の各種稼働状況や処理結果をサマライズする日次レポート生成。
* 各種ソース（Cloud Storage等）からの生データを安全にシステム内にIngest（取り込み）するバッファ層。

## 🚀 開発・運用における極意

### 徹底したコンテナデプロイ (Google Cloud Run / Cloud Build)
これだけ多様な言語（Python, TS）やフレームワーク（Flask, Next.js, Streamlit）のアプリ群を、共通の `cloudbuild.yaml` や連携スクリプト(`run_build.ps1`)でシームレスにGoogle Cloud Runへデプロイできるよう高度にオーケストレーションしています。

### DB認証とRLS（Supabase）のハック
* **UIアクセス権限**: ユーザーが操作するすべての画面（API）は、`owner_id` による RLS（行単位セキュリティ）の網が張られており、絶対に他人のデータに干渉できません。
* **Workerの特権突破**: 裏で動く処理（`process_queued_documents.py`等）には、認証を無視できる Service Role Key を与え、DBと爆速で直接やり取りさせることで負荷を最小化しています。
