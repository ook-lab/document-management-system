# Stage H: 構造化（Structuring）

あなたは非構造化テキストから構造化データを抽出する専門家です。

## 入力情報

- **ファイル名**: {file_name}
- **ドキュメントタイプ**: {doc_type}
- **ワークスペース**: {workspace}
- **現在日付**: {current_date}

## テキスト

```
{combined_text}
```

## タスク

このテキストから、以下の情報を抽出してJSON形式で出力してください。

### 抽出項目

1. **document_date** (文字列, YYYY-MM-DD形式, または null)
   - ドキュメントの基準日（作成日、発行日、イベント日など）
   - 例: "2025-03-15"

2. **tags** (文字列の配列)
   - 検索用のタグ 5-10個
   - カテゴリ、トピック、キーワードなど
   - 例: ["会議資料", "プロジェクトA", "予算"]

3. **metadata** (オブジェクト)
   - ドキュメントタイプに応じた追加情報
   - 自由形式だが、有用な情報を含めること

## 出力形式

```json
{
  "document_date": "2025-03-15",
  "tags": ["タグ1", "タグ2", "タグ3"],
  "metadata": {
    "キー1": "値1",
    "キー2": "値2"
  }
}
```

## 重要な注意事項

- **必ずJSON形式で出力してください**（```json ブロック内に記述）
- document_date が不明な場合は null を使用
- tags は必ず配列形式で出力
- metadata には、テキストから抽出できる有用な情報を含める
- 推測や想像で情報を追加しないこと（テキストに明記されている情報のみ抽出）
